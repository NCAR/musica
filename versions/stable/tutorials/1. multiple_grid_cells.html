Multiple Grid Cells in MUSICA#

In MUSICA, the State object that is present within the Solver object has an attribute called number_of_grid_cells. This attribute dictates the number of independent sets of well-mixed air masses whose chemical system will be solved by the same numerical solver. This tutorial will go over a simple example of solving a multi-grid-cell chemical system in MUSICA.

MUSICA: Before Getting Started#

It is heavily recommended to go through the tutorials in the MusicBox repository first before going through the MUSICA ones since the former explains the workflow used in the latter. However, if you are only interested in MUSICA, here’s how you can set up a virtual environment for it:

conda create --name musica python=3.9
conda activate musica
pip install musica
conda install ipykernel scikit-learn seaborn scipy dask

1. Importing Libraries#

Below is a list of the required libraries for this tutorial:

[1]:

import musica
import musica.mechanism_configuration as mc
import matplotlib.pyplot as plt
import pandas as pd
import numpy as np
pd.set_option('display.float_format', str) # This is done to make the arrays more readable
np.set_printoptions(suppress=True) # This is done to make the arrays more readable

2. Defining a System#

This code snippet is a MUSICA version of setting up a system, which has an identical workflow to MusicBox. For an explanation of this code in MusicBox, please refer to the Basic Workflow tutorial in the MusicBox repository.

[2]:

A = mc.Species(name="A") # Create each of the species with their respective names
B = mc.Species(name="B")
C = mc.Species(name="C")
species = [A, B, C] # Bundle the species into a list
gas = mc.Phase(name="gas", species=species) # Create a gas phase object containing the species

r1 = mc.Arrhenius( # Create the reactions with their name, constants, reactants, products, and phase
    name="A_to_B",
    A=4.0e-3,  # Pre-exponential factor
    C=50,      # Activation energy (units assumed to be K)
    reactants=[A],
    products=[B],
    gas_phase=gas
)

r2 = mc.Arrhenius(
    name="B_to_C",
    A=4.0e-3,
    C=50,
    reactants=[B],
    products=[C],
    gas_phase=gas
)

mechanism = mc.Mechanism( # Define the mechanism which contains a name, the species, the phases, and reactions
    name="musica_micm_example",
    species=species,
    phases=[gas],
    reactions=[r1, r2]
)

3. Creating the Solver#

Something more unique to MUSICA is that you have to manually define the solver. A solver integrates the chemical reactions that determine how atmospheric chemistry proceeds over time. There are a handful of solvers available, but Rosenbrock Standard Order will be used here. For more information on the types of solvers available, go here.

[3]:

solver = musica.MICM(mechanism = mechanism, solver_type = musica.SolverType.rosenbrock_standard_order)

4. Creating the State#

For MUSICA, the state must be created manually as well, with the number of grid cells being passed into the create_state() function. The state represents everything pertinent to solve chemistry. By solving, we meaning determining the concentrations at the next time step. Feel free to change the num_grid_cells value to experiment yourself.

[4]:

num_grid_cells = 2
state = solver.create_state(num_grid_cells)

5. Populating the Grid Cells#

5 dimensions will be used to populate the data for each of the air masses:

  • temperature (Kelvin),

  • pressure (Pascals), and

  • the concentrations of each of the species (mol/m3).

The NumPy array has the reshape() function called on it so that it can be split up in the next step. The two arguments specify the number of rows and columns of the array, where -1 means any number of rows. Do note that the ordering inside the array matters and cannot be changed.

[5]:

box_model_values = np.array([[300, 101253.3, 5, 5, 5], [100, 11253.3, 20, 3, 7]])
box_model_values = box_model_values.reshape(-1, 5)
display(box_model_values)

6. Splitting up the Array Output#

Next, the values from the box_model_values array are taken and populated into variables so that they can be passed into the solver’s state. The sample is organized into 5 columns that represent the 5 variables. The sample’s rows each represent a distinct grid cell. These columns are populated into their respective variables and then passed into the solver’s state. Do note that set_conditions() and set_concentrations() must be called with their respective arguments for the solver to successfully run. The concentrations must go through the extra step of being bundled into a dictionary since MUSICA explicitly requires a dictionary argument in the set_concentrations function. Lastly, an empty array is initialized to represent the solved concentration array at every time step, as well as the time step length, simulation length, and the current time step (all in seconds).

[6]:

temperatures = box_model_values[:, 0]
pressures = box_model_values[:, 1]
concentrations = {
    "A": [],
    "B": [],
    "C": []
}
concentrations["A"] = box_model_values[:, 2]
concentrations["B"] = box_model_values[:, 3]
concentrations["C"] = box_model_values[:, 4]

state.set_conditions(temperatures, pressures)
state.set_concentrations(concentrations)
concentrations_solved = []
time_step_length = 1
sim_length = 60
curr_time = 0

7. Running the Solver#

This code solves the system at every specified time step and the solved concentrations are appended to the array. The first step will always be the initial conditions since at time = 0 seconds the reaction has not begun.

[7]:

while curr_time <= sim_length:
    solver.solve(state, curr_time)
    concentrations_solved.append(state.get_concentrations())
    curr_time += time_step_length

8. Preparing the Results (Advanced; Optional Read)#

Here, a new array is made that grabs only the first value (grid cell) for each key (species) at every time step. The time step index is divided by the time_step_length to account for time step lengths that are greater than one for proper array indexing. That new array is then passed into a Pandas DataFrame with the concentration columns renamed. Next, a time column is created from a range that represents the elapsed time at each time step. In this simulation, the temperature, pressure, and air density are all constant, so numpy’s repeat() function is used to repeat their respective values for every time step. Once all the attributes are added to the DataFrame, their order is changed to follow a more logical flow. Due to the complexity of this code cell, it has been bundled into a function that takes in an argument for the grid cell you wish to convert to a DataFrame. The function is then called twice, one for each grid cell index.

[8]:

def convert_results_single_cell(cell_index):
    concentrations_solved_pd = []
    for i in range(0, sim_length + 1, time_step_length):
        concentrations_solved_pd.append({species: concentration[cell_index] for species, concentration in concentrations_solved[int(i/time_step_length)].items()})
    df = pd.DataFrame(concentrations_solved_pd)
    df = df.rename(columns = {'A' : 'CONC.A.mol m-3', 'B' : 'CONC.B.mol m-3', 'C' : 'CONC.C.mol m-3'})
    df['time.s'] = list(map(float, range(0, sim_length + 1, time_step_length)))
    df['ENV.temperature.K'] = np.repeat(temperatures[cell_index], sim_length/time_step_length + 1.0)
    df['ENV.pressure.Pa'] = np.repeat(pressures[cell_index], sim_length/time_step_length + 1.0)
    df['ENV.air number density.mol m-3'] = np.repeat(state.get_conditions()['air_density'][cell_index], sim_length/time_step_length + 1.0)
    df = df[['time.s', 'ENV.temperature.K', 'ENV.pressure.Pa', 'ENV.air number density.mol m-3', 'CONC.A.mol m-3', 'CONC.B.mol m-3', 'CONC.C.mol m-3']]
    return concentrations_solved_pd, df

[9]:

concentrations_solved_pd_0, df_0 = convert_results_single_cell(0)
concentrations_solved_pd_1, df_1 = convert_results_single_cell(1)
